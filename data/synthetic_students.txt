================================================================================
SYNTHETIC PROTOTYPING DATA — FABRICATED, NOT REAL MIDDLEBURY DATA
================================================================================
Every name below is invented. Every course ID, title, meeting block, and
capacity is invented. None of this describes real Middlebury students,
courses, enrollment numbers, or schedules. It exists only so that Milestone 1
(build the student-course bipartite graph) has something concrete to load.

Per Part VII of the project prompt, real data (if you obtain it later, e.g.
from the registrar or a survey you run) must be kept in a separate file and
clearly labeled "REAL DATA" so the two are never confused in the write-up.

Generated: 2026-07-17

This file is the human-readable reference copy. The same data, in a form
that's actually pleasant to parse with fscanf()/strtok() in C, lives in:
  data/courses.csv       (id,department,title,block,capacity)
  data/preferences.csv   (name,class_year,major,rank1..rank10)
If you ever edit one, edit the other to match — nothing auto-syncs them.
================================================================================
